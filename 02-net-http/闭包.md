# Go 闭包

## 一、闭包定义

### 1. 官方定义

闭包是一个函数值，它引用了其函数体外部的变量 —— 该函数可以访问并修改其引用的外部变量，即使该函数在其外部变量的作用域之外执行。

### 2. 通俗理解

闭包 = 函数 + 函数能访问的 “外部变量环境”，像一个 “带背包的函数”，背包里装着它需要的外部变量，走到哪带到哪。

## 二、核心特性

### 1. 捕获外部变量

闭包会 “绑定” 其定义时所在作用域的外部变量，而非复制变量的值 —— 修改闭包内的外部变量，会影响原变量。

```
func counter() func() int {
    count := 0 // 外部变量
    return func() int { // 闭包捕获count
        count++
        return count
    }
}

// 调用示例
c1 := counter()
fmt.Println(c1()) // 1
fmt.Println(c1()) // 2（count被持续修改）
```

### 2. 延长变量生命周期

- 普通局部变量：函数执行完后存于栈中，出作用域即销毁；
- 闭包捕获的变量：编译器会将其从**栈逃逸到堆**，只要闭包还被引用，堆上的变量就不会被 GC 回收。

### 3. 私有性

闭包捕获的变量仅能被闭包访问，外部无法直接修改，实现变量 “私有”。

## 三、关键原理

| 变量存储位置 | 生命周期           | 闭包影响                     |
| :----------- | :----------------- | :--------------------------- |
| 栈（Stack）  | 函数执行完立即销毁 | 捕获后逃逸到堆，延长生命周期 |
| 堆（Heap）   | 无引用时被 GC 回收 | 闭包持有引用，变量持续存在   |
| 全局区       | 程序全程存在       | 闭包≠全局变量，仅按需存在    |

## 四、典型应用场景

### 1. 中间件（最常用）

封装通用逻辑（日志、鉴权、限流），不修改核心业务代码，实现功能增强（如 HTTP 日志中间件）：

```
func loggerMiddleware(next http.Handler) http.Handler {
    // 闭包捕获外部变量next（核心业务处理器）
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        start := time.Now()
        log.Printf("Started %s %s", r.Method, r.URL.Path)
        next.ServeHTTP(w, r) // 执行核心业务
        log.Printf("Completed in %v", time.Since(start))
    })
}
```

- 核心：`next` 本是栈变量，被闭包捕获后逃逸到堆，只要闭包（`loggedHandler`）存在，`next` 就不会销毁。

### 2. 函数工厂

动态生成具有特定行为的函数（如定制化计数器）。

### 3. 延迟执行

延迟访问 / 修改外部变量（如 goroutine 中访问循环变量）。

## 五、易混点 & 注意事项

### 1. 闭包 vs 全局变量

| 特性     | 闭包捕获的变量 | 全局变量     |
| :------- | :------------- | :----------- |
| 作用域   | 仅闭包可访问   | 全局可访问   |
| 生命周期 | 闭包引用时存在 | 程序全程存在 |
| 内存占用 | 按需回收       | 持续占用     |

### 2. 常见坑

- 循环中创建闭包，易捕获循环变量的最终值（需通过参数传值规避）；
- 闭包持有外部变量引用，若长期不释放可能导致内存泄漏。

## 六、核心总结

1. 闭包是 “带外部变量环境的函数”，可捕获、修改外部变量；
2. 核心价值：解耦逻辑、实现变量私有、延长变量生命周期；
3. 关键机制：捕获的变量从栈逃逸到堆，由 GC 管理生命周期；
4. 典型场景：HTTP 中间件、函数工厂、定制化逻辑封装。
