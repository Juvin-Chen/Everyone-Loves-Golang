# Gin 高级架构：中间件流水线与上下文传值 (Context)

## 一、 核心顿悟：为什么带参中间件和结构体 Handler 不冲突？

在工业级代码中，我们通常会同时使用这两种模式，因为它们关注的**“维度”**完全不同，属于完美正交（互不干扰）的关系。

### 1. 角色分工明确

- **中间件（带参闭包工厂）：** 负责**外围协议层 (HTTP/Web)**。就像高档餐厅门口的保安，负责检查请柬（Token）、VIP等级。因为保安不需要炒菜，只需要接收上级的指令（比如只放行 "VIP"），所以用**“闭包传参”**最轻便。
- **Handler（结构体方法）：** 负责**内部业务层 (Logic/Database)**。就像后厨的大厨。大厨不关心客人是怎么进来的，他只关心怎么用手里的工具（依赖注入的 `repo` 数据库实例）把菜做好。

### 2. 代码组装的画面感

在路由注册时，它们就像套娃一样完美嵌套。中间件在外挡风遮雨，Handler 在内安心干活：

```
// 外部套一层中间件（闭包），处理 Web 请求和拦截
// 内部调用结构体 Handler，处理核心业务逻辑
r.POST("/messages", AuthMiddleware("VIP"), myStructHandler.Create)
```

## 二、 进阶实战：保安如何把信息传递给大厨？

在这个流水线中，经常会遇到一个极其关键的场景：

**中间件（保安）在门口解析了请求头里的 Token，查到了这个客人的真实用户 ID 是 `1001`。内部的 Handler（大厨）在存数据库时，正好需要用到这个 `1001`。**

既然它们是两个独立的函数，保安怎么把这个 `1001` 传给大厨呢？

答案是：使用 Gin 的核心神器 —— **`\*gin.Context` 里的 `Set` 和 `Get` 方法**。

### 1. 上游（中间件）：使用 `c.Set()` 贴便利贴

保安查完信息后，不直接喊话，而是在当前这个 HTTP 请求的“包裹”上贴一张内部便利贴。

```
// 1. 带参中间件工厂
func AuthMiddleware(role string) gin.HandlerFunc {
    return func(c *gin.Context) {
        token := c.GetHeader("Authorization")
        
        // 假设这里去校验了 token，并解析出了 userID 是 1001
        userID := 1001 
        
        // 💡 核心传值：把数据写入当前请求的上下文中
        // "userID" 是我们自己约定的暗号（Key）
        c.Set("userID", userID)
        
        c.Next() // 放行，交给下一个处理者
    }
}
```

### 2. 下游（结构体 Handler）：使用 `c.Get()` 撕便利贴

大厨接手这个请求时，直接从包裹上撕下便利贴，提取里面的信息。

```
// 2. 结构体 Handler
type MessageHandler struct {
    repo repository.MessageRepo
}

func (h *MessageHandler) Create(c *gin.Context) {
    // 💡 核心取值：从上下文中读取上游贴上的暗号
    value, exists := c.Get("userID")
    if !exists {
        c.JSON(500, gin.H{"error": "系统异常，找不到用户信息"})
        return
    }
    
    // ⚠️ 必考考点：类型断言 (Type Assertion)
    // 因为 c.Get 返回的是 any (interface{}) 类型，Go 不知道它原来是啥。
    // 我们必须手动告诉 Go：“我知道它是个 int，帮我转回来！”
    uid := value.(int) 
    
    // 拿到确定的 uid 后，就可以用结构体里的 repo 去干活了
    h.repo.AddMessage(uid, "这是留言内容")
    
    c.JSON(200, gin.H{"msg": "发布成功"})
}
```

## 三、 完整架构全景图

将依赖注入、闭包中间件、上下文传值结合在一起，这就是一个标准的工业级 Gin 路由长相：

```
func SetupRouter() *gin.Engine {
    r := gin.Default()

    // 1. 依赖注入：造好工具，交给大厨
    repo := repository.NewMySQLRepo() 
    msgHandler := handler.NewMessageHandler(repo) 

    // 2. 路由分组与保安上岗
    vipGroup := r.Group("/vip")
    
    // 3. 完美结合：保安接指令("VIP")，大厨拿数据(c.Get)并用工具(repo)干活
    vipGroup.Use(AuthMiddleware("VIP")) 
    {
        vipGroup.POST("/messages", msgHandler.Create) 
    }

    return r
}
```
