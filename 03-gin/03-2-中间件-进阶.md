
# Gin 中间件进阶：带参中间件与工厂模式

## 一、 为什么我们需要“进阶版”？（解决什么痛点）

在基础版中，中间件的签名被 Gin 死死限制住了，必须是：
`func(c *gin.Context)`
**痛点来了：** 既然参数只能是 `c *gin.Context`，那我怎么给中间件传我们自己定义的参数呢？
* 比如：我想写一个限流中间件，有的接口每秒允许 10 次访问，有的允许 100 次。我怎么把 `10` 和 `100` 传进去？
* 基础版做不到！为了解决这个问题，我们需要**引入“返回函数的函数”（即工厂模式/闭包）**。

---

## 二、 进阶版的核心骨架（牢记这个模板）

进阶版的本质是：**外层函数接收参数，内层函数执行逻辑。**

```go
// 外层：工厂函数，名字随便起。负责接收你想要的任何参数。
// 它的返回值类型必须是 gin.HandlerFunc
func MyAdvancedMiddleware(param1 string, param2 int) gin.HandlerFunc {
    
    // 1. 【初始化区域】：这里的代码只在项目启动（路由注册）时执行一次！
    log.Printf("中间件已加载，参数为: %s, %d\n", param1, param2)
    
    // 内层：真正的中间件逻辑
    return func(c *gin.Context) {
        // 2. 【运行区域】：这里的代码，每次有 HTTP 请求过来时都会执行！
        
        // 闭包魔法：这里可以直接使用外层的 param1 和 param2
        if param2 > 10 {
             // do something...
        }
        
        c.Next()
    }
}
```

## 三、 实战对比：基础版 vs 进阶版

我们以“**角色权限校验 (Role Auth)**”为例，看看进阶版有多香。

### 基础版的笨办法：代码冗余
因为不能传参，只能为每一个角色写一个死函数：
```go
// 只能写死给 Admin 验证
func AdminAuth(c *gin.Context) {
    if c.GetHeader("Role") != "Admin" { c.Abort(); return }
    c.Next()
}

// 只能写死给 VIP 验证
func VIPAuth(c *gin.Context) {
    if c.GetHeader("Role") != "VIP" { c.Abort(); return }
    c.Next()
}

// 注册
r.GET("/admin", AdminAuth, handler)
r.GET("/vip", VIPAuth, handler)
```

### 进阶版的优雅解法：一劳永逸
写一个可以接收 `requiredRole` 参数的中间件：
```go
// AuthMiddleware 接收一个字符串参数，返回一个 Gin 处理器
func AuthMiddleware(requiredRole string) gin.HandlerFunc {
    return func(c *gin.Context) {
        // 获取当前请求用户的角色
        userRole := c.GetHeader("Role") 
        
        // 判断当前角色是否等于外层传进来的 requiredRole
        if userRole != requiredRole {
            c.JSON(403, gin.H{"error": "权限不足，你需要：" + requiredRole})
            c.Abort()
            return
        }
        // 权限通过，放行
        c.Next()
    }
}

// 注册时，按需传入参数，就像拼乐高一样！
r.GET("/admin", AuthMiddleware("Admin"), handler)
r.GET("/vip", AuthMiddleware("VIP"), handler)
r.GET("/user", AuthMiddleware("User"), handler)
```

## 四、 核心考点：代码执行的时机（面试常问）

理解进阶中间件，一定要分清“**什么时候执行**”。

```go
func SetupConfigMiddleware(dbName string) gin.HandlerFunc {
    // 【阶段 A】代码在这里：
    // 当你写下 r.Use(SetupConfigMiddleware("my_db")) 时，
    // 这里立刻执行！且整个程序生命周期只执行这一次！
    // 适合做：初始化数据库连接、加载配置文件、打印启动日志。
    
    return func(c *gin.Context) {
        // 【阶段 B】代码在这里：
        // 只有当用户真的用浏览器访问了网页，触发了这个路由，
        // 这里的代码才会执行！每天有一万个人访问，就执行一万次！
        // 适合做：校验 Token、计算请求时间、修改 Header。
        c.Next()
    }
}
```
