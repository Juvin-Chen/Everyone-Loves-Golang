# 终极：如何避免写无数个 Context Wrapper 函数？

如果你要在 Context 里传 10 种不同的数据，千万不要写 10 对 `Set/Get` 函数！在真实的商业项目中，我们主要用以下两种极其优雅的方式来解决。

## 方案一：打包成“结构体”法（业务开发最常用、最推荐）

**核心思想：不要一次传一件衣服、一条裤子，直接打包塞进一个“行李箱”里传过去。**

在中间件解析出 Token 后，我们通常不会只拿到一个 ID，还会拿到用户名、角色等一堆信息。我们直接定义一个 `UserSession` 结构体，把它作为一个整体存进去。

### 1. 定义行李箱（结构体）
```go
// 这就是我们的行李箱
type UserSession struct {
    UserID   int
    UserName string
    Role     string
    IsAdmin  bool
}
```

### 2. 只需要写这一对终极 Setter/Getter
```go
const sessionKey = "app_user_session"

// 存行李箱
func SetUserSession(c *gin.Context, session UserSession) {
    c.Set(sessionKey, session)
}

// 取行李箱
func GetUserSession(c *gin.Context) (UserSession, bool) {
    val, exists := c.Get(sessionKey)
    if !exists {
        return UserSession{}, false // 返回空结构体和 false
    }
    
    // 整个项目只需要在这里做一次类型断言！
    session, ok := val.(UserSession) 
    return session, ok
}
```

### 3. 大厨（Handler）爽到飞起的使用体验
```go
func (h *MessageHandler) Create(c *gin.Context) {
    // 一次提取，要啥有啥！
    session, ok := ctxutil.GetUserSession(c)
    if !ok {
        c.JSON(401, gin.H{"error": "未登录"})
        return
    }

    // 直接用点(.)操作符取值，绝对类型安全，且自带代码提示！
    h.repo.AddMessage(session.UserID, session.UserName, "内容")
}
```
**优点**：高度内聚，极度清晰。这一个 Wrapper 就可以搞定 99% 的业务传值需求。

## 方案二：Go 1.18+ 泛型法（框架开发者最爱）

**核心思想：用魔法打败魔法，写一个支持任意类型的万能工具。**

如果你用的 Go 版本 >= 1.18，我们可以使用**泛型 (Generics)**。这就像是造了一个自动适应形状的超级模具。

### 1. 编写泛型 Wrapper
注意看函数名后面的 `[T any]`，这就是告诉 Go 语言：“T 可以是任何类型，你调用的时候传什么，T 就是什么。”

```go
// 泛型 Setter
// val T 表示 val 可以是任意类型
func Set[T any](c *gin.Context, key string, val T) {
    c.Set(key, val)
}

// 泛型 Getter
// 返回值也是 T 类型
func Get[T any](c *gin.Context, key string) (T, bool) {
    val, exists := c.Get(key)
    if !exists {
        var zero T // 返回该类型的零值
        return zero, false
    }

    // 泛型类型断言
    typedVal, ok := val.(T)
    return typedVal, ok
}
```

### 2. 在业务中的使用方法
```go
// 保安存值时：自动推导出是 int 和 string
ctxutil.Set(c, "USER_ID", 1001)
ctxutil.Set(c, "USER_ROLE", "Admin")

// ==========================================

// 大厨取值时：手动告诉泛型函数，我这次要取的是啥类型
// 如果你告诉它是 int，它返回给你的 uid 就是妥妥的 int！
uid, ok := ctxutil.Get[int](c, "USER_ID") 

role, ok := ctxutil.Get[string](c, "USER_ROLE")
```
**优点**：真正的一劳永逸，写一遍代码，可以存取全世界所有的类型。

---

### 总结与建议

* 如果你是在**写具体的业务模块**（比如处理用户的登录信息），**毫不犹豫地选方案一（结构体法）**。因为把相关信息打包在一个结构体里，代码可读性最高。
* 如果你是在**封装一个给全公司用的底层基础库**，希望它能处理各种乱七八糟的场景，**选方案二（泛型法）**。
