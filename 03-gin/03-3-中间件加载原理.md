#  Gin 中间件「启动时自动打印」**底层原理逐行拆解**
（纯代码执行逻辑，不讲空话，100%讲透为什么启动就打印）

## 核心前提（必须先懂）
`SetupConfigMiddleware` **不是Gin的特殊魔法**，它就是一个**普普通通的 Go 函数**！
它的执行规则，和你写 `fmt.Println()` 完全一样：**调用即执行**。

---

## 一、终极答案：为什么启动就打印？
### 代码执行顺序规则（Go 语言固定规则）
当你写这行代码时：
```go
r.Use( SetupConfigMiddleware("my_db") )
```

**Go 语言会先计算括号里的参数 → 先执行 `SetupConfigMiddleware("my_db")`**
→ 函数体内的代码**立刻、马上、同步执行**
→ 打印语句直接运行
→ 执行完后，把**返回值**传给 `r.Use()`

**整个过程发生在「程序启动阶段」，和「用户请求」没有半毛钱关系！**

---

## 二、逐行模拟程序启动（你跟着走一遍就懂）
我们把代码拆碎，一步一步看执行流程：

### 你的代码
```go
func SetupConfigMiddleware(dbName string) gin.HandlerFunc {
    // 🔥 这行打印代码
    fmt.Println("【初始化】正在连接数据库:", dbName)

    // ... 其他初始化代码

    // 返回一个匿名函数（真正的中间件）
    return func(c *gin.Context) {
        c.Next()
    }
}

func main() {
    r := gin.Default()

    // 关键代码！程序启动执行这一行
    r.Use(SetupConfigMiddleware("my_db"))

    r.Run(":8080")
}
```

---

### 程序启动，执行 `main` 函数，步骤如下：
#### 第 1 步：执行 `r := gin.Default()`
创建 Gin 引擎， nothing else。

#### 第 2 步：执行 `r.Use( SetupConfigMiddleware("my_db") )`
**Go 语言的执行逻辑：**
1. **先算括号里的参数**：必须先把 `SetupConfigMiddleware("my_db")` 执行完，拿到返回值，才能传给 `r.Use()`
2. **调用函数**：立刻进入 `SetupConfigMiddleware` 函数内部
3. **执行函数内代码**：
   ```go
   // 这行代码直接运行！！！
   fmt.Println("【初始化】正在连接数据库: my_db")
   ```
   ✅ **这就是你看到的「启动瞬间打印」的根本原因！**
4. 函数执行完毕，返回一个匿名函数（阶段B的中间件）
5. `r.Use()` 只做一件事：**把这个匿名函数存起来**（等到用户请求时再用）

#### 第 3 步：执行 `r.Run(":8080")`
启动服务，等待用户请求。

---

## 三、一句话说穿本质
```go
// 这不是注册，这是【直接调用函数】！
SetupConfigMiddleware("my_db")  

// 函数调用 = 函数体内代码立刻执行
// 打印 = 函数调用时同步执行的结果
```

**Gin 根本没插手这个打印！**
是 **你自己调用了函数**，函数自己打印的！

---

## 四、两个阶段的代码边界（一眼分清）
```go
func SetupConfigMiddleware(dbName string) gin.HandlerFunc {
    // ======================================
    // 🔥 这里是【函数本身】
    // 调用函数时 → 立刻执行 → 启动时执行
    // 打印、初始化数据库都在这里
    // ======================================
    fmt.Println("启动时打印！") 

    // ======================================
    // 返回的【匿名函数】
    // 只是被存起来，启动时不执行！
    // 只有用户请求来了，才会执行
    // ======================================
    return func(c *gin.Context) {
        fmt.Println("请求时打印！") 
        c.Next()
    }
}
```

---

# 五、最直白的总结
1. **`SetupConfigMiddleware(...)` 是函数调用**
2. **函数调用 → 函数体立刻执行**
3. **打印语句在函数体内 → 启动瞬间就打印**
4. **返回的匿名函数，只是被Gin存起来，请求来了才跑**

这就是为什么：**程序一启动，还没任何人访问，就已经打印了！**
