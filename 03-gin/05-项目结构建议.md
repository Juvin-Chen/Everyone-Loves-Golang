# 承接05-分组路由和项目结构.go
## 4. 项目结构建议

当项目变大时，将所有路由和处理器放在 `main.go` 会难以维护。推荐使用以下结构：

```
myapp/
├── cmd/
│   └── server/
│       └── main.go           # 程序入口
├── internal/
│   ├── handler/              # 处理器（业务逻辑）
│   │   ├── user.go
│   │   └── product.go
│   ├── middleware/           # 中间件
│   │   ├── auth.go
│   │   └── logger.go
│   ├── model/                # 数据模型
│   │   └── user.go
│   ├── repository/           # 数据访问层（可选）
│   └── service/              # 服务层（可选）
├── pkg/                      # 可复用的公共代码
├── config/                   # 配置文件
├── go.mod
└── go.sum
```

**示例：handler/user.go**
```go
package handler

import (
    "github.com/gin-gonic/gin"
    "net/http"
)

type UserHandler struct {
    // 可以注入 service 等
}

func NewUserHandler() *UserHandler {
    return &UserHandler{}
}

func (h *UserHandler) List(c *gin.Context) {
    c.JSON(http.StatusOK, gin.H{"users": []string{"alice", "bob"}})
}

func (h *UserHandler) Get(c *gin.Context) {
    id := c.Param("id")
    c.JSON(http.StatusOK, gin.H{"id": id, "name": "user"})
}
```



**cmd/server/main.go**

```go
package main

import (
    "myapp/internal/handler"
    "myapp/internal/middleware"

    "github.com/gin-gonic/gin"
)

func main() {
    r := gin.Default()

    // 全局中间件
    r.Use(middleware.Logger())

    // 路由分组
    api := r.Group("/api")
    {
        v1 := api.Group("/v1")
        {
            userHandler := handler.NewUserHandler()
            v1.GET("/users", userHandler.List)
            v1.GET("/users/:id", userHandler.Get)
        }
    }

    r.Run(":8080")
}
```

## 5. 使用 `gin.H` 与结构体返回

返回 JSON 时，可以使用 `gin.H`（map）快速构造，也可以返回结构体（自动 JSON 序列化）。

```go
type UserResponse struct {
    ID   int    `json:"id"`
    Name string `json:"name"`
}

func (h *UserHandler) Get(c *gin.Context) {
    user := UserResponse{ID: 1, Name: "Alice"}
    c.JSON(http.StatusOK, user)
}
```

## 6. 项目初始化与模块化

推荐将路由注册分离到单独的函数中，便于测试和复用。

**internal/router/router.go**


```go
package router

import (
    "myapp/internal/handler"
    "myapp/internal/middleware"
    "github.com/gin-gonic/gin"
)

func Setup() *gin.Engine {
    r := gin.Default()
    r.Use(middleware.Logger())

    // 健康检查
    r.GET("/health", func(c *gin.Context) {
        c.JSON(200, gin.H{"status": "ok"})
    })

    api := r.Group("/api")
    {
        v1 := api.Group("/v1")
        {
            userHandler := handler.NewUserHandler()
            v1.GET("/users", userHandler.List)
            v1.GET("/users/:id", userHandler.Get)
        }
    }
    return r
}
```



**cmd/server/main.go**


```go
package main

import (
    "myapp/internal/router"
    "log"
)

func main() {
    r := router.Setup()
    if err := r.Run(":8080"); err != nil {
        log.Fatal("Failed to start server: ", err)
    }
}
```

## 7. 配置文件与环境变量

可以使用 `viper` 等库管理配置。简单场景可直接用环境变量。

```go
port := os.Getenv("PORT")
if port == "" {
    port = "8080"
}
r.Run(":" + port)
```
